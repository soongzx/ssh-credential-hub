## 1. 测试策略

### 1.1 测试金字塔

```
        ┌─────────┐
        │   E2E   │  ← 少量关键流程
       ┌──────────┐
       │ Integration│ ← 模块间交互
      ┌───────────┐
      │   Unit    │ ← 核心逻辑覆盖
     ┌─────────────┐
```

### 1.2 测试优先级

| 优先级 | 说明 | 覆盖率目标 |
|--------|------|-----------|
| P0 | 核心功能（CRUD、数据库） | 100% |
| P1 | 重要功能（IPC、UI 交互） | 80% |
| P2 | 一般功能（筛选、搜索） | 60% |
| P3 | 辅助功能（导入导出） | 40% |

### 1.3 每次发布前必须通过

- [ ] 单元测试：核心 Repository、Service
- [ ] 集成测试：IPC 通信流程
- [ ] E2E 测试：核心用户路径

## 9. 测试框架配置

### 9.1 技术栈

- **测试框架**: Vitest
- **断言库**: Vitest 内置 expect
- **环境**: Node.js

### 9.2 测试命令

```bash
npm run test        # 运行测试（watch 模式）
npm run test:run    # 运行测试（单次）
npm run test:ui     # 运行测试（带 UI）
```

### 9.3 测试文件结构

```
tests/
├── testUtils.ts                    # 测试工具
├── connectionRepository.test.ts     # 连接 Repository 测试
├── tagRepository.test.ts           # 标签 Repository 测试
├── groupRepository.test.ts        # 分组 Repository 测试
└── cryptoService.test.ts          # 加密服务测试（待实现）
```

### 9.4 测试数据隔离

- 每个测试用例使用独立的数据库实例
- 测试完成后自动清理数据库文件
- 测试数据库文件位于 `data/test/` 目录
